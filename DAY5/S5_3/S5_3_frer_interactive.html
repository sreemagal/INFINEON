FRER — Frame Replication and Elimination for Reliability

IEEE 802.1CB · R-TAG sequence numbering · disjoint path redundancy · sequence recovery

How FRER works end-to-end: The talker's sequence-generation function assigns a sequence number to each frame and inserts an R-TAG carrying that number. The frame is replicated onto two or more disjoint network paths. At the listener, the sequence-recovery function accepts the first copy of each sequence number and silently discards all subsequent duplicates. A single physical path failure — cable cut, switch crash, transient error — cannot interrupt the stream because the other path continues delivering.
R-TAG frame insertion:  [Dst MAC | Src MAC | 0xF1C1 | seq_num (16b) | reserved (16b) | EtherType | Payload | FCS*]
Sequence number: 16-bit, wraps 0–65535 · R-TAG removed by sequence-recovery function before delivery to application
