Automotive Ethernet security and Media Access Control Security (MACsec)

Media Access Control Security (MACsec) is a link-layer security engine for Ethernet. It is not merely encryption. It provides data-origin authenticity, integrity, replay protection, and optional confidentiality for frames on a protected Layer 2 (L2) link.

This artifact explains the architecture, the protected Ethernet Protocol Data Unit (PDU), the MACsec Key Agreement (MKA) control plane, message sequences, and automotive design trade-offs such as generalized Precision Time Protocol (gPTP) inline protection versus bypass.

1Data-communications context first

The correct starting point is not “should we encrypt?” The correct starting point is “what can an attacker do on or near this Ethernet link?”

Passive attack

An attacker taps a camera link or gateway uplink and reads unprotected traffic. Confidentiality mode prevents useful disclosure of the protected payload.

Active attack

An attacker injects a spoofed control or state frame. MACsec validation detects frames that are not protected with the active Secure Association (SA).

Replay attack

An attacker records a valid old protected frame and resends it later. Packet Number (PN) replay checks reject stale frames outside the configured replay window.

Important framing: automotive Ethernet security is about confidentiality, integrity, authenticity, replay resistance, segmentation, and survivability after partial compromise. Secrecy alone is not enough.

2The MACsec architecture

MACsec uses a small set of architecture objects. Expanding the acronyms early prevents later confusion.

Connectivity Association (CA)

The overall security relationship among participants that are allowed to communicate securely on a Local Area Network (LAN).

Secure Channel (SC)

A directional secure traffic relationship inside a Connectivity Association. One transmitter has one transmit Secure Channel.

Secure Association (SA)

A concrete cryptographic context inside a Secure Channel. Each Secure Association uses one Secure Association Key (SAK).

MAC Security Entity (SecY)

The port-side MACsec engine. It protects transmitted frames and validates received frames.

3Complete MACsec Protocol Data Unit format structure

A MACsec-protected Ethernet frame remains an Ethernet frame. The major structural change is that a Secure Tag (SecTAG) is inserted after the source MAC address and an Integrity Check Value (ICV) is appended at the end.

VLAN placement note: the standards-oriented teaching model places the Secure Tag immediately after the source MAC address. That means the original user-data region after the Secure Tag is protected. Some system designs intentionally expose an outer Virtual Local Area Network tag for switching or provider bridging. Automotive designs must state the MAC Security Entity placement explicitly because it determines whether VLAN Identifier (VID) and Priority Code Point (PCP) are visible to intermediate forwarding logic.

4MACsec Key Agreement and Extensible Authentication Protocol over LAN PDU structure

MACsec Key Agreement is the control plane. It discovers authenticated peers, elects a Key Server, distributes the Secure Association Key, and keeps Secure Associations fresh.

Connectivity Association Key (CAK)

A long-term pre-shared or Extensible Authentication Protocol (EAP) derived secret used as the root for MKA operations.

Integrity Check Key (ICK) and Key Encryption Key (KEK)

The Integrity Check Key authenticates MKA Protocol Data Units. The Key Encryption Key protects distributed Secure Association Key material.

Secure Association Key (SAK)

The data-plane key installed in the MAC Security Entity for one Secure Association.

6Cipher suites and protection modes

MACsec always targets authenticity and integrity. Confidentiality is optional and is selected by mode and cipher-suite behavior.

Galois/Counter Mode - Advanced Encryption Standard with 128-bit key (GCM-AES-128)

The mandatory baseline family for ordinary MACsec support. It combines Advanced Encryption Standard counter-mode encryption with Galois-field authentication.

GCM-AES-256 and Extended Packet Number (XPN) variants

GCM-AES-256 increases key size. Extended Packet Number variants reduce rekey pressure by extending the packet numbering space for very high-speed or long-lived links.

7Automotive deployment choices

A realistic vehicle does not simply “turn on encryption everywhere.” It defines link protection, bypass exceptions, startup behavior, diagnostics, and timing assumptions.

Protect by default

Use MACsec on exposed or mission-critical Ethernet links, especially camera-to-switch, zonal-controller uplinks, gateway-to-central-compute links, and high-value sensor paths.

Mode per traffic class

Choose integrity-only when visibility is needed but tamper detection is critical. Choose confidentiality plus integrity when information disclosure is also unacceptable.

Bypass as exception

Keep MKA on the Uncontrolled Port. Treat diagnostics, update, and timing bypasses as explicit exceptions with compensating controls.

8Generalized Precision Time Protocol and MACsec

Generalized Precision Time Protocol can be designed to work with MACsec inline, but timing accuracy and one-step timestamping make this an engineering-sensitive area.

Inline generalized Precision Time Protocol with MACsec

Works best with two-step timing and bounded MACsec latency variation. Sync and Follow_Up semantics avoid the need to rewrite a protected one-step timestamp inside an authenticated frame.

Generalized Precision Time Protocol bypass

May be used when one-step assumptions, startup constraints, or hardware limitations make inline protection unacceptable. It must be treated as a real security exception.

9Elucidative automotive scenarios

Scenario A: frame injection on a zonal uplink

Attack

A compromised service device injects a spoofed Ethernet frame toward a zonal controller. The frame has a believable source MAC address and VLAN Identifier, but no valid Secure Tag and Integrity Check Value for the active Secure Association.

MACsec result

The receiving MAC Security Entity cannot validate the frame. It discards the frame and increments validation-failure counters. Higher-layer control software never sees the spoofed payload.

Scenario B: replay of a valid old camera frame

Attack

An attacker records a previously valid MACsec-protected camera frame and retransmits it later.

MACsec result

The Secure Tag contains an old Packet Number. The receive replay window rejects it before the old image payload can be accepted.

Scenario D: VLAN visibility decision

Virtual Local Area Network tag inside protected data

The VLAN Identifier and Priority Code Point are protected from link attackers, but intermediate switches that are not MACsec endpoints cannot use them for forwarding or traffic-class selection.

Outer Virtual Local Area Network tag in clear

An intentionally exposed outer tag can support switching and queue selection before MACsec termination, but it becomes visible metadata and must be part of the security model.

10Short recap

Protocol essentials

  • Media Access Control Security is Layer 2 hop-by-hop link protection.
  • The MAC Security Entity protects transmit frames and validates received frames.
  • Connectivity Association defines the secure relationship, Secure Channel defines a directional channel, and Secure Association defines the active keyed cryptographic context.
  • Secure Tag carries Tag Control Information, Association Number, Short Length, Packet Number, and optional Secure Channel Identifier.
  • Integrity Check Value proves authenticity and integrity. Confidentiality is optional.

Automotive essentials

  • Use MACsec on critical or exposed Ethernet links.
  • Use Pre-Shared Key style MKA when deterministic startup matters.
  • Keep MKA on the Uncontrolled Port because it establishes MACsec.
  • Make generalized Precision Time Protocol bypass a documented exception, not a default assumption.
  • Track counters and logs because validation failures are security diagnostics.
