Differential Manchester Encoding — Polarity Independence

Why reversing the cable polarity does not corrupt the decoded data

The differential property: In DME, bit values are encoded in the presence or absence of a transition at the start of each period — not in the absolute signal level. This means the receiver never needs to know which wire carries the logical HIGH. If the cable is wired backwards, the waveform is inverted but the transition pattern is identical, and every bit is decoded correctly without any re-configuration.

What changes when polarity is inverted

The absolute signal levels are flipped — what was HIGH becomes LOW and vice versa. The waveform shape looks like a mirror image of the original. An oscilloscope on the inverted cable shows a different voltage pattern.

What does not change

The position of every transition stays exactly the same. The count of transitions per period stays exactly the same. The presence or absence of a start-of-period transition — the data encoding — stays exactly the same.

NRZ — polarity dependent

If you reverse an NRZ cable, every bit flips: all 1s become 0s and all 0s become 1s. The data is completely corrupted. The receiver has no way to detect or correct this without an out-of-band signal or a known preamble.

DME — polarity independent

Reversing a DME cable has no effect on the decoded data. The receiver decodes by counting transitions per period, which is unaffected by inversion. No preamble, no handshake, and no configuration change is needed.
Practical consequence: In systems using DME (token ring, ISDN, some industrial fieldbuses), the installer does not need to identify which terminal of a twisted pair is the positive and which is the negative conductor. Either orientation works. This simplifies cabling, eliminates a class of installation errors, and means a cable fault that reverses polarity does not silently corrupt data — the link continues operating correctly.
